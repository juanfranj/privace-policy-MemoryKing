2. Information We Collect

2.1 Information Stored Locally on Your Device

MemoryKing stores the following information locally on your device only. This data is never transmitted to our servers or any third parties:

  • Game Progress: Levels completed, stars earned, best scores, and best completion times
  • User Preferences: Sound settings, music settings, language selection, and notification preferences
  • Timestamps: Last time each level was played

This data is stored using Android's Room database and DataStore, and remains private to your device. When you uninstall the app, all this data is permanently deleted.

2.2 Information Collected by Third-Party Services

MemoryKing uses third-party services to provide functionality and improve the app experience. These services may automatically collect certain information:

Google AdMob (Advertising)

We use Google AdMob to display advertisements. AdMob automatically collects:

  • IP address - Used to determine general location and serve region-appropriate ads
  • Device identifiers - Android Advertising ID (a unique, user-resettable ID for advertising)
  • Device information - Model, operating system version, screen size, and screen resolution
  • Ad interaction data - Which ads you view, click, and how long you view them
  • Diagnostic information - App crashes related to ads, ad load times, for fraud prevention

Purpose: To serve relevant advertisements and prevent ad fraud.

Data Transmission: All data is encrypted in transit using TLS (HTTPS).

Your Control: You can opt out of personalized ads in your Android device settings (Settings > Google > Ads > Opt out of Ads Personalization) or reset your advertising ID at any time.

Learn more: Google AdMob Privacy Policy

Firebase Analytics

We use Firebase Analytics to understand how users interact with MemoryKing and improve the app. Firebase automatically collects:

  • App usage data - App opens, session duration, screen views
  • Navigation patterns - Which screens you visit and in what order
  • Device information - Model, OS version, screen resolution, language settings
  • General location - Country-level location derived from IP address (not precise location)
  • User engagement metrics - How long you play, which levels you complete
  • First-party identifiers - Anonymous app instance IDs

Purpose: To improve app performance, understand user behavior, fix bugs, and enhance user experience.

Data Retention: Firebase Analytics data is automatically deleted after 14 months.

Learn more: Firebase Privacy and Security

Firebase Crashlytics

We use Firebase Crashlytics to identify and fix app crashes. Crashlytics collects:

  • Crash logs and stack traces - Technical information about what caused the crash
  • Device state at crash time - Memory usage, battery level, network connectivity
  • Custom logs - Technical logs we've added to help debug issues

Purpose: To identify crashes and improve app stability.

Data Retention: Crash data is retained for 90 days.

Learn more: Firebase Privacy and Security

2.3 Permissions We Request

10.1 Types of Tracking Technologies Used

  • First-party identifiers: Firebase generates anonymous app instance IDs to track app sessions and analytics.

  • Device identifiers: Android Advertising ID (AAID) is used by AdMob for advertising purposes. This is a unique, user-resettable identifier.
